# Welcome to FloImg

**FloImg** is a universal image workflow engine that provides three core operations—**generate**, **transform**, **save**—that work consistently across JavaScript, CLI, YAML, and MCP.

## The Problem FloImg Solves

**LLMs are non-deterministic.** Great for creativity, terrible for "resize to exactly 800x600."

**Image workflows are fragmented.** AI images need the OpenAI SDK. Transforms need Sharp. Charts need Chart.js. Each has different APIs with no way to chain them.

**FloImg solves both.** Deterministic execution with a unified abstraction. LLMs handle natural language; FloImg handles precise execution.

## Quick Example

```typescript
import createClient from '@teamflojo/floimg';
import openai from '@teamflojo/floimg-openai';

const floimg = createClient();
floimg.registerGenerator(openai({ apiKey: process.env.OPENAI_API_KEY }));

// Generate → Transform → Save
const image = await floimg.generate({
  generator: 'openai',
  params: { prompt: 'A product photo of headphones', size: '1024x1024' }
});

const resized = await floimg.transform(image, {
  op: 'resize',
  params: { width: 800 }
});

await floimg.save(resized, 's3://bucket/product.png');
```

## Interfaces

FloImg provides multiple interfaces for different use cases:

| Interface | Best For |
|-----------|----------|
| **SDK** | TypeScript/JavaScript applications |
| **CLI** | Shell scripts, CI/CD pipelines |
| **YAML** | Configuration-driven workflows |
| **MCP** | AI agents (Claude, etc.) |
